STEP 01

PPT 구성안 Text 출력

소스 문서를 분석하여 40장 분량의 Master Script Report를 생성하는 프롬프트

1Content Architecture Prompt
# Role: Chief Content Architect
Task: Analyze ALL uploaded sources and generate a consistent 40-page [Master Script Report].

## [Variables: Please Fill Below]
- Target Audience: << 의사결정 임원 >>
- Presentation Objective: << 신사업에 진입해야 하는가? 진입 한다면 어떻게?  돈을 벌 수 있나? >>

## Instruction Guidelines
1. 소스 문서의 핵심 팩트와 데이터를 통합하여 논리적 흐름(서론-본론-결론)을 구축
1-1 신사업 기업 Dynamics
1-2 신사업의 KSF
1-3 신사업 진입 장벽
1-4 수익성?
2. 지정된 [Target Audience]의 수준과 관심사에 맞춘 전문적인 용어와 설득력 있는 문체 사용
3. McKinsey, BCG, Bain의 문제해결 Framework, Issue Tree, 5 Whys, MECE 활용
4. Why 왜 해야하는가, What 무엇을 해야 하는가?, How 어떻게 해야하는가? Where 타겟 시장과 우리 위치는 어디인가?
5. Devil's Advocate을 전체 구조를 챌린지 하고, 논리를 보완하여 전체를 다시 구성
5-1 Outdated Data 삭제와 권위있는 Source를 활용하여, 논리를 탄탄하게 다시 구성

## Output Format (Strictly Follow)
슬라이드 번호: (1~40)
제목: (해당 페이지의 핵심 헤드라인)
화면 텍스트: (핵심 데이터 및 키워드 10줄 이상 요약), 정량 Data, 숫자, 통계는 Footer에 Source 출력 (발표 기관과 발표 날짜 포함)
Footer: 상단 숫자, 통계에 대한 Source와 발표 날짜
STEP 02-1

슬라이드 렌더링 시스템

Visual Identity, Typography, Layout 규칙 및 실행 스크립트를 포함한 디자인 시스템 프롬프트

2-1Adaptive Design System Prompt
[Adaptive Presentation Design System]
1. Visual Identity
- Background: 무조건 아무것도 없는 완전한 흰색 단색 배경(Solid White, #FFFFFF). 배경에 격자 무늬(Grid), 워터마크, 배경 이미지 등은 절대 표출 금지
- Theme: Top-tier Consulting Firm Style (McKinsey, BCG, Bain)
- Palette: BG[#FFFFFF], Text[#111111], Accent[#1428A0]
- Font: 모든 텍스트는 반드시 'SamsungOneKorean700' 또는 'SamsungOneKorean500' 폰트만 사용 (모든 장표에서 폰트 종류와 크기가 달라지면 안 되며 일관성 유지)
- Graphics: Sharp rectangles, blue gradients for shapes, thin grey vertical/horizontal dividers, strict margins. Bottom area graphics must be schematized and explicitly display data/numbers

2. Typography & Writing Rules (글자 크기 및 문체 일관성 엄수)
- 장표마다 글자 크기가 달라지면 안됨, 아래 규칙을 모든 장표에 일관되게 유지할 것
- Main Title: 상단 중앙 정렬, 30pt
- Subtitle: 14pt
- Body Content (소제목 아래 본문): 10pt
- Footer: 출처(Data sources)와 발표/발간 날짜는 반드시 슬라이드 가장 밑바닥에 9pt로 표기
- Writing Style (매우 중요):
  ▶ 길이 제한: 상세 내용 및 불릿 포인트(·)는 무조건 2줄 이내로 간결하게 요약해서 출력
  ▶ 개조식 작성: 서술형 문장 금지. "가능합니다" -> "가능", "진행했습니다" -> "진행"과 같이 명사형 또는 간단한 요약형(개조식) 문구로 작성
  ▶ 마침표 삭제: 모든 문장의 끝에는 마침표(.) 절대 사용 금지

3. Dynamic Layout Rules (논리적 흐름에 맞춰 A, B, C 중 택 1)
- Type A(Hierarchy): Focus on text structure. Top-center aligned main titles(30pt), solid blue subtitle bars(14pt), bulleted body(10pt)
- Type B(Data/Metrics): Maximize chart area. Schematized bottom diagrams explicitly displaying data/metrics. Clean headers, high-contrast metric callouts using Accent color
- Type C(Split View): Mid-line vertical divider. Left side for context/lists(10pt), right side for process steps, visual comparisons, or schematized data

4. Execution
Strictly follow the slide count and content boundaries. Construct the presentation strictly following the logical flow and structured layout styles of McKinsey, BCG, and Bain. Assign Layout A, B, or C to logical sections ensuring maximum contrast and visual hierarchy.

  "
}

1. 논리 구조: MECE(상호배제와 전체포괄) 원칙에 입각하여 단순 요약이 아닌 컨설팅 보고서처럼 상세하고 논리적으로 탄탄하게 전개

[Slide 번호]
▶ Layout: Type A / B / C 중 택 1
▶ Main Title (30pt): 슬라이드 핵심 메시지 요약
▶ Subtitle (14pt): 소제목 내용
▶ Body Content (10pt):
   - 본문 내용 1 (컨설팅 보고서 수준의 논리 전개, 2줄 이내 요약, 개조식, 마침표 생략)
   - 본문 내용 2 (소스 데이터 매칭 및 최신 데이터 백업, 2줄 이내 요약, 개조식, 마침표 생략)

▶ Footer (9pt): Source: 권위 있는 데이터 출처 및 발표 날짜

[Execution Rules: 중요 실행 규칙]
1. 업로드된 소스 데이터와 1:1로 매칭하여 누락 없이 반영
