[Adaptive Presentation Design System] 1. Visual Identity - Background (최우선 절대 원칙): 어떠한 예외 상황(특정 내용 강조, 다크 모드, 차트 등)에서도 배경색 변경이나 반전 절대 금지. 40장 모든 장표의 배경은 무조건 아무것도 없는 100% 순백색 단색(Solid White, #FFFFFF)으로 강제 고정. 배경에 옅은 색상, 그라데이션, 격자 무늬(Grid), 워터마크, 배경 이미지는 일절 표출 금지. - Theme: Top-tier Consulting Firm Style (McKinsey, BCG, Bain) - Palette: BG[#FFFFFF], Text[#111111], Accent[#1428A0] - Font(최우선 절대 원칙): 모든 텍스트는 반드시 'SamsungOneKorean700' 폰트만 사용. (모든 장표에서 폰트 종류와 크기가 달라지면 안 되며 일관성 유지) - Graphics: Sharp rectangles, blue gradients for shapes, thin grey vertical/horizontal dividers, strict margins. Bottom area graphics must be schematized and explicitly display data/numbers. 2. Typography & Writing Rules (최우선 절대 원칙 : 글자 크기 및 문체 일관성 엄수) - 장표마다 글자 크기가 달라지면 안됨, 아래 규칙을 모든 장표에 일관되게 유지할 것. - Main Title: 상단 중앙 정렬, 30pt. - Subtitle: 14pt. - Body Content (소제목 아래 본문): 10pt. - Footer: 출처(Data sources)와 발표/발간 날짜는 반드시 슬라이드 가장 밑바닥 왼쪽에 7pt로 표기. (권위있는 외부기관만 표시, 없으면 출력금지) 출력 양식 Source : 기관, 리포트 제목 ('2x.x월), 외부기관, 뉴스 기사 ('25.2월) - Writing Style (매우 중요): ▶ 길이 제한: 상세 내용 및 불릿 포인트(·)는 무조건 2줄 이내로 간결하게 요약해서 출력. ▶ 개조식 작성: 서술형 문장 금지. "가능합니다" -> "가능", "진행했습니다" -> "진행"과 같이 명사형 또는 간단한 요약형(개조식) 문구로 작성. ▶ 마침표 삭제: 모든 문장의 끝에는 마침표(.) 절대 사용 금지. ▶ Executive Summary, BLUF, Pyramid, Issue Tree, Devil's Advocate, MECE 등 컨설팅 용어는 출력하지 말고, 바로 핵심 내용만 출력 3. Dynamic Layout Rules (논리적 흐름에 맞춰 A, B, C 중 택 1) - Type A(Hierarchy): Focus on text structure. Top-center aligned main titles(30pt), solid blue subtitle bars(14pt), bulleted body(10pt). - Type B(Data/Metrics): Maximize chart area. Schematized bottom diagrams explicitly displaying data/metrics. Clean headers, high-contrast metric callouts using Accent color. - Type C(Split View): Mid-line vertical divider. Left side for context/lists(10pt), right side for process steps, visual comparisons, or schematized data. 4. Execution Strictly follow the slide count and content boundaries. Construct the presentation strictly following the logical flow and structured layout styles of McKinsey, BCG, and Bain.
